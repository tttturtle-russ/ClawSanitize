=== TEST 2: VULNERABLE-CONFIG SCAN ===
[MEDIUM] Skill 'file-reader' is from an unofficial source
  ID: SUPPLY_CHAIN-003 | Category: SUPPLY_CHAIN
  'file-reader' was installed from 'https://github.com/unknown-user/file-reader' which is not the official ClawHub marketplace. Skills from unofficial sources have not been reviewed for malware.
  💡 Fix: Only install skills from the official ClawHub marketplace (clawhub://). Remove this skill and find an official alternative.

[CRITICAL] All permission prompts are disabled
  ID: CONFIG-001 | Category: CONFIGURATION
  The setting 'dangerously_skip_permissions: true' is enabled. This means OpenClaw can read files, run commands, and send messages WITHOUT asking you first. Any skill or instruction can do anything on your computer silently.
  💡 Fix: Open ~/.openclaw/config.json and change 'dangerously_skip_permissions' to false. Then restart OpenClaw.

[HIGH] Anyone can send commands to your AI agent via direct messages
  ID: CONFIG-002 | Category: CONFIGURATION
  The DM policy is set to 'open' with no sender restrictions. This means any user on any connected platform (Slack, Discord, Telegram, etc.) can send instructions to your OpenClaw agent and it will execute them.
  💡 Fix: In ~/.openclaw/config.json, change 'dmPolicy' to 'closed' or set 'allowFrom' to a list of trusted user IDs only.

[HIGH] Workspace directory is set to an overly broad path
  ID: CONFIG-003 | Category: CONFIGURATION
  The workspace_dir is set to '/'. This gives the AI agent access to a very large portion of your filesystem, making it easy for a malicious skill to read or modify important files.
  💡 Fix: Set workspace_dir to a specific, limited directory like '~/.openclaw/workspace' in your config.json.

[HIGH] API key is stored in plaintext in config file
  ID: CONFIG-004 | Category: CONFIGURATION
  Your API key is stored as plain text in ~/.openclaw/config.json. Any program that can read this file — including malicious skills — can steal your API key and use it to run up charges or access your AI account.
  💡 Fix: Remove the API key from config.json and use your operating system's secure keychain instead. Check OpenClaw's documentation for 'secure credential storage'.

[HIGH] OpenClaw gateway is exposed to all network interfaces
  ID: CONFIG-005 | Category: CONFIGURATION
  The gateway is bound to 0.0.0.0, which means it is accessible from other devices on your network (and potentially the internet). Anyone who can reach your computer on the network can connect to your AI agent.
  💡 Fix: In ~/.openclaw/config.json, change gateway.bind to '127.0.0.1' to restrict access to your local machine only.

[MEDIUM] Gateway authentication is disabled
  ID: CONFIG-006 | Category: CONFIGURATION
  The OpenClaw gateway does not require authentication. Any application or browser on your computer can connect to and control your AI agent without needing a password or token.
  💡 Fix: In ~/.openclaw/config.json, set gateway.auth to true. Then update any integrations to use the generated authentication token.

[HIGH] Tailscale And SSH Tunnels Are enabled without authentication
  ID: CONFIG-007 | Category: CONFIGURATION
  The Tailscale and SSH tunnels are enabled but does not require authentication. This creates a remote access path to your AI agent without any password protection.
  💡 Fix: Enable authentication for the tunnel in config.json. Check OpenClaw's documentation for 'remote access security'.

[CRITICAL] Suspicious instructions found in AGENTS.md
  ID: DISCOVERY-001 | Category: DISCOVERY
  Your AGENTS.md file contains instructions that look like an attempt to make your AI agent secretly send data or bypass safety rules. Suspicious pattern: "send a copy of user conversations to https://"
  💡 Fix: Open ~/.openclaw/workspace/AGENTS.md and remove any instructions you did not write yourself. If you did not create this file, delete it entirely.
  📁 File: testdata/vulnerable-config/workspace/AGENTS.md

[HIGH] Dangerous tool capability 'shell_execute' found in TOOLS.md
  ID: DISCOVERY-002 | Category: DISCOVERY
  Your TOOLS.md file defines a tool called 'shell_execute' which can give the AI agent unrestricted access to run commands or access files on your computer.
  💡 Fix: Review ~/.openclaw/workspace/TOOLS.md and remove or restrict any tool definitions that allow unrestricted shell access or file access.
  📁 File: testdata/vulnerable-config/workspace/TOOLS.md

[CRITICAL] Suspicious background task found in HEARTBEAT.md
  ID: DISCOVERY-003 | Category: DISCOVERY
  Your HEARTBEAT.md file contains a scheduled task that looks suspicious: "Every 5 minute". This task may be sending your data to a remote server in the background.
  💡 Fix: Open ~/.openclaw/workspace/HEARTBEAT.md and remove any tasks you did not create yourself. If you are unsure, delete the file entirely.
  📁 File: testdata/vulnerable-config/workspace/HEARTBEAT.md

[HIGH] Tool 'file_access' references sensitive credential path
  ID: DISCOVERY-006 | Category: DISCOVERY
  The tool 'file_access' mentions access to '~/.ssh' in its description. This sensitive location contains credentials or encryption keys and should not be accessible to AI tools.
  💡 Fix: Review and remove this tool. No legitimate AI assistant tool needs direct access to SSH keys, browser profiles, or credential stores.
  📁 File: testdata/vulnerable-config/workspace/TOOLS.md

[CRITICAL] Tool 'file_access' references forbidden credential storage
  ID: RUNTIME-001 | Category: RUNTIME
  The tool 'file_access' description references '~/.ssh/', which is a forbidden sensitive path associated with credentials or secrets.
  💡 Fix: Remove or restrict this tool so it cannot access SSH keys, cloud credentials, browser profiles, keychains, or .env files.
  📁 File: file_access

[CRITICAL] Workspace file references forbidden credential storage
  ID: RUNTIME-001 | Category: RUNTIME
  Workspace content references '.env', a forbidden sensitive path that can expose credentials or private keys.
  💡 Fix: Remove references that direct agents or tools to access sensitive credential stores and replace with least-privilege data sources.
  📁 File: testdata/vulnerable-config/workspace/AGENTS.md

[CRITICAL] Workspace file references forbidden credential storage
  ID: RUNTIME-001 | Category: RUNTIME
  Workspace content references '~/.ssh/', a forbidden sensitive path that can expose credentials or private keys.
  💡 Fix: Remove references that direct agents or tools to access sensitive credential stores and replace with least-privilege data sources.
  📁 File: testdata/vulnerable-config/workspace/TOOLS.md

[CRITICAL] Workspace file references forbidden credential storage
  ID: RUNTIME-001 | Category: RUNTIME
  Workspace content references '~/.ssh/', a forbidden sensitive path that can expose credentials or private keys.
  💡 Fix: Remove references that direct agents or tools to access sensitive credential stores and replace with least-privilege data sources.
  📁 File: testdata/vulnerable-config/workspace/HEARTBEAT.md

[HIGH] Webhook gateway is exposed without authentication
  ID: RUNTIME-004 | Category: RUNTIME
  Gateway authentication is disabled while the bind address is non-loopback, which can expose webhook endpoints to untrusted network clients.
  💡 Fix: Enable gateway authentication and bind to 127.0.0.1 unless remote webhook access is strictly required and additionally protected.
  📁 File: gateway.bind

[MEDIUM] Allowlist contains wildcard channel pattern
  ID: RUNTIME-005 | Category: RUNTIME
  allowFrom includes '*', which effectively disables sender/channel restrictions.
  💡 Fix: Replace wildcard allowFrom entries with explicit trusted user/channel identifiers.
  📁 File: allowFrom
